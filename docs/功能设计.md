# 网页阅读辅助与 AI 对话插件 (WebReader AI) 功能设计说明书.

![1774248801082](image/功能设计/1774248801082.png)

## 一、 产品概述

本插件是一款基于 Chrome Extension Manifest V3 标准开发的浏览器辅助工具。旨在通过多种灵活的网页内容提取技术（智能 DOM 解析、长图截取等），结合多模型 AI 驱动的对话能力，帮助用户在不同站点上实现自动化、沉浸式的高效信息获取与处理。

## 二、 技术架构与依赖

* **运行环境：** Chromium 浏览器 (Manifest V3 规范)
* **核心通信：** `chrome.runtime` (Background Service Worker 与 Content Script / Popup 通信)
* **数据存储：** `chrome.storage.local` (存储大容量对话与缓存) / `chrome.storage.sync` (存储跨设备用户配置)
* **核心依赖库：**
  * `Readability.js`: 用于精准提取网页正文与结构化 DOM。
  * `pdf.js` (可选/按需加载): 用于处理本地上传的 PDF 并转为图片或文本供视觉模型分析。

## 三、 基础功能 (Basic Features)

### 3.1 模型配置与管理 (Model Management)

支持多模型提供商接入，允许用户根据具体任务（如长文总结、视觉分析）按需动态切换模型。

* **支持提供商：** OpenAI, Qwen (通义千问), Anthropic (Claude), Ollama (本地大模型) 等。
* **关键特性：**
  * 支持自定义 BaseURL 和 API Key。
  * 模型能力标识（明确该模型是否支持 Vision 视觉读取、是否支持 Tool/Function Calling 推理）。
  * 支持全局默认流式输出 (Stream) 与超时等待配置。

### 3.2 AI 对话 (AI Chat)

借鉴 NextChat 的优秀交互设计，提供侧边栏或独立窗口的沉浸式对话体验。

* **关键特性：**
  * **多对话管理：** 按会话 (Session) 隔离历史记录，支持按站点自动归档。
  * **多模态输入：** 支持文本输入、图片上传、以及 PDF 上传（自动切片转图片或提取文本）。
  * **上下文控制：** 自动携带当前网页的核心提取内容作为 System/User Prompt 的前置上下文。

## 四、 核心功能 (Core Features)

### 4.1 角色管理 (Role Management)

用于管理应对不同网页场景的系统提示词 (System Prompts)。

* **关键特性：**
  * 预设与自定义角色（如：“学术论文翻译官”、“电商数据提取专家”、“代码审查员”）。
  * 每个角色包含独立的 RoleName、标识 Icon 以及核心 Prompts。

### 4.2 网页阅读与分析 (Web Reading & Analysis)

插件的核心“感知”模块，负责将非标准化的网页转化为 AI 可理解的数据。

* **分析方法分类：**
  1. **智能分析 (Readability)：** 预处理网页（破解 CSS 截断、点击“展开”按钮、自动滚动加载）后，提取纯净 HTML/文本。适合新闻、博客、文档。
  2. **长图分析 (Screenshot)：** 针对防抓取严格或强依赖视觉排版的页面，通过 API 截取完整长图，交由具备 Vision 能力的模型处理。
  3. **混合分析 (Hybrid)：** 同时提取关键 DOM 结构和关键区域截图。
* **子功能：网页切换 (自动化执行)**
  * 支持配置“动作脚本 (Action Script)”或 CSS 选择器组。插件可自动寻找页面上的菜单树、下一页按钮，自动点击并等待加载，实现跨页面的连续阅读与分析。

### 4.3 站点配置与管理 (Site Rules & Routing)

插件的“智能路由中心”。根据用户当前所在的网站，自动分发对应的处理策略。

* **匹配规则优先级：**
  1. **按具体网页名 (Site Name)：** 最高优先级。通过读取 `<meta property="og:site_name">` 或解析 `<title>` 获取名称（如“知乎”、“GitHub”），实现对非技术用户的友好配置。
  2. **按域名/路径 (Domain/URL Prefix)：** 次优先级。精准匹配特定网址。
  3. **按正则通配符 (Regex)：** 兜底策略，匹配一类复杂的 URL 规则。
* **策略绑定：** 匹配成功后，自动为该页面分配【默认角色】、【默认模型】和【默认解析方法】。

## 五、 数据模型设计 (Data Models)

以下使用 TypeScript 接口风格定义核心数据结构，便于直接映射到数据库或 Storage 存储：

**TypeScript**

```
// ==========================================
// 1. 模型配置数据 (Models)
// ==========================================
interface ModelConfig {
  modelId: string;             // 唯一标识 (如 UUID)
  provider: 'OpenAI' | 'Qwen' | 'Anthropic' | 'Ollama' | 'Custom';
  modelName: string;           // 实际调用的模型名 (如 gpt-4o)
  baseUrl: string;             // 接口地址 (Ollama 等必填)
  apiKey: string;              // 密钥 (本地存储需加密)
  capabilities: {
    supportVision: boolean;    // 是否支持长图/图片分析
    supportTool: boolean;      // 是否支持推理/函数调用
  };
  stream: boolean;             // 是否默认流式输出
  timeout: number;             // 超时等待时间，默认 60000 (ms)
}

// ==========================================
// 2. 角色管理数据 (Roles)
// ==========================================
interface RoleConfig {
  roleId: string;
  roleName: string;            // 角色名称
  icon: string;                // UI 展示图标 (Emoji 或 URL)
  prompts: string;             // 核心 System Prompt
  parameters?: object;         // 可选：绑定的模型参数 (如 temperature)
}

// ==========================================
// 3. 站点配置数据 (SiteConfigs)
// ==========================================
interface SiteConfig {
  siteId: string;
  matchType: 'siteName' | 'domain' | 'url_prefix' | 'regex'; // 匹配类型
  matchRule: string;           // 具体的规则 (如 "知乎", "*.github.com")
  defaultRoleId: string;       // 关联的默认角色 ID
  defaultModelId: string;      // 关联的默认模型 ID
  analysisMethod: 'readability' | 'screenshot' | 'hybrid'; // 默认网页解析方法
  autoActions?: AutoAction[];  // 可选：自动化网页切换/点击行为
}

interface AutoAction {
  action: 'click' | 'scroll' | 'wait';
  selector?: string;           // 目标 DOM 选择器
  value?: number;              // 滚动距离或等待时间(ms)
}

// ==========================================
// 4. 对话与消息数据 (Chat)
// ==========================================
interface ChatSession {
  sessionId: string;
  siteId?: string;             // 关联的站点，用于隔离对话
  title: string;
  modelId: string;             // 当前会话所用模型
  roleId: string;              // 当前会话所用角色
  createdAt: number;           // 时间戳
}

interface ChatMessage {
  messageId: string;
  sessionId: string;
  role: 'system' | 'user' | 'assistant';
  content: string;             // 文本内容
  attachments?: Attachment[];  // 多模态附件
  createdAt: number;
}

interface Attachment {
  type: 'image' | 'pdf_converted_image' | 'file';
  dataUrl: string;             // Base64 或本地 Object URL
}

// ==========================================
// 5. 网页阅读结果数据 (PageContent)
// ==========================================
interface PageContent {
  url: string;
  title: string;
  siteName: string;            // 提取出的网站名称
  textContent: string;         // Readability 提取的纯文本 (用于计算 Token)
  htmlContent: string;         // Readability 提取的纯净 HTML
  screenshotData?: string;     // 长图 Base64 数据 (如触发长图分析)
  attachments: Attachment[];   // 网页中提取的关键附件/图片
}
```
